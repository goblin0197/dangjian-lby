# Git未提交接口文档总结

## 概述
本文档总结了当前git仓库中未提交的所有接口变更，包括新增和修改的接口。

## 主要新增和修改的模块

1. **公告管理模块增强** - 新增阅读状态管理功能
2. **公告扩展功能模块** - 新增附件管理功能  
3. **材料管理模块** - 新增完整的材料提交、审核、归档、模板管理功能
4. **量化考核模块** - 新增量化数据和指标管理功能

## 接口详情

### 1. 公告管理模块 (NoticeController - 已修改)

#### 新增功能
- **标记公告为已读** (`POST /notice/markRead`)
  - 功能：用户标记公告为已读状态
  - 权限：所有用户
  - 参数：`announcementId` (公告ID)

- **获取未读公告数量** (`GET /notice/unreadCount`)
  - 功能：获取当前用户的未读公告数量
  - 权限：所有用户
  - 参数：无

### 2. 公告扩展功能模块 (新增 - AnnouncementExtensionController)

#### 附件管理
- **获取公告附件列表** (`GET /announcement/attachment/list`)
  - 功能：获取指定公告的附件列表
  - 权限：所有用户
  - 参数：`announcementId` (公告ID)

- **添加公告附件** (`POST /announcement/attachment/add`)
  - 功能：为公告添加附件
  - 权限：超级管理员、组织管理员
  - 参数：`AnnouncementAttachment` 对象

- **删除公告附件** (`DELETE /announcement/attachment/delete/{id}`)
  - 功能：删除指定附件
  - 权限：超级管理员、组织管理员
  - 参数：`id` (附件ID)

### 3. 材料归档查询模块 (新增 - MaterialArchiveController)

#### 基础功能
- **获取归档材料列表** (`GET /api/material/archive/list`)
  - 功能：查询归档材料列表
  - 权限：所有用户
  - 参数：`MaterialArchiveQueryRequest` 查询条件

- **查看归档详情** (`GET /api/material/archive/detail`)
  - 功能：查看归档材料详情
  - 权限：所有用户
  - 参数：`id` (归档ID)

- **归档材料** (`POST /api/material/archive/archive`)
  - 功能：将终审通过的材料归档
  - 权限：超级管理员、组织管理员
  - 参数：`submissionId` (提交ID), `archiveRemark` (归档备注)

- **下载归档材料文件** (`GET /api/material/archive/download`)
  - 功能：下载归档材料文件
  - 权限：所有用户
  - 参数：`id` (归档ID)

- **批量导出归档材料** (`POST /api/material/archive/batchExport`)
  - 功能：批量导出归档材料为压缩包
  - 权限：超级管理员、组织管理员
  - 参数：`MaterialArchiveBatchExportRequest` 导出请求

### 4. 材料提交审核模块 (新增 - MaterialSubmissionController)

#### 提交管理
- **获取材料提交列表** (`GET /api/material/submission/list`)
  - 功能：查询材料提交列表
  - 权限：所有用户
  - 参数：`MaterialSubmissionQueryRequest` 查询条件

- **提交材料** (`POST /api/material/submission/submit`)
  - 功能：提交材料审核
  - 权限：所有用户
  - 参数：`MaterialSubmissionSubmitRequest` 提交请求

#### 审核流程
- **审核材料（初审）** (`PUT /api/material/submission/audit`)
  - 功能：初审材料
  - 权限：超级管理员、组织管理员
  - 参数：`MaterialSubmissionAuditRequest` 审核请求

- **退回材料** (`PUT /api/material/submission/reject`)
  - 功能：退回材料
  - 权限：超级管理员、组织管理员
  - 参数：`id` (提交ID), `auditOpinion` (审核意见)

### 5. 材料模板管理模块 (新增 - MaterialTemplateController)

#### 模板管理
- **获取材料模板列表** (`GET /api/material/template/list`)
  - 功能：查询材料模板列表
  - 权限：所有用户
  - 参数：`MaterialTemplateQueryRequest` 查询条件

- **添加材料模板** (`POST /api/material/template/add`)
  - 功能：添加材料模板
  - 权限：超级管理员、组织管理员
  - 参数：`MaterialTemplateAddRequest` 添加请求

- **更新材料模板** (`PUT /api/material/template/update`)
  - 功能：更新材料模板
  - 权限：超级管理员、组织管理员
  - 参数：`MaterialTemplateUpdateRequest` 更新请求

- **删除材料模板** (`DELETE /api/material/template/delete`)
  - 功能：删除材料模板
  - 权限：超级管理员、组织管理员
  - 参数：`DeleteRequest` 删除请求

### 6. 量化数据模块 (新增 - QuantifyDataController)

#### 数据管理
- **获取数据列表** (`GET /quantify/data/list`)
  - 功能：查询量化数据列表
  - 权限：超级管理员、组织管理员
  - 参数：`period` (周期，可选)

- **根据目标获取数据** (`GET /quantify/data/target`)
  - 功能：根据目标获取数据
  - 权限：所有用户
  - 参数：`targetId` (目标ID), `targetType` (目标类型), `period` (周期)

- **添加数据** (`POST /quantify/data/add`)
  - 功能：添加量化数据
  - 权限：超级管理员
  - 参数：`QuantifyData` 数据对象

- **批量添加数据** (`POST /quantify/data/batchAdd`)
  - 功能：批量添加量化数据
  - 权限：超级管理员
  - 参数：`List<QuantifyData>` 数据列表

### 7. 量化指标模块 (新增 - QuantifyIndicatorController)

#### 指标管理
- **获取指标列表** (`GET /quantify/indicator/list`)
  - 功能：查询量化指标列表
  - 权限：超级管理员、组织管理员
  - 参数：`status` (状态，可选)

- **添加指标** (`POST /quantify/indicator/add`)
  - 功能：添加量化指标
  - 权限：超级管理员
  - 参数：`QuantifyIndicator` 指标对象

- **更新指标** (`PUT /quantify/indicator/update`)
  - 功能：更新量化指标
  - 权限：超级管理员
  - 参数：`QuantifyIndicator` 指标对象

- **删除指标** (`DELETE /quantify/indicator/delete/{id}`)
  - 功能：删除量化指标
  - 权限：超级管理员
  - 参数：`id` (指标ID)

- **切换指标状态** (`PUT /quantify/indicator/toggle/{id}`)
  - 功能：切换指标状态
  - 权限：超级管理员
  - 参数：`id` (指标ID), `status` (状态)

## 权限控制说明

所有接口都配置了适当的权限控制，主要权限角色包括：

- **超级管理员** (SUPER_ADMIN_ROLE) - 最高权限，可管理所有功能
- **组织管理员** (ORG_ADMIN_ROLE) - 可管理本组织相关功能
- **所有用户** - 无特殊权限要求，可访问基础功能

## 技术规范

- 所有接口遵循RESTful设计规范
- 使用Swagger进行API文档管理
- 统一的响应格式：`BaseResponse<T>`
- 统一的错误处理机制
- 完整的参数校验

这些接口为党建系统提供了完整的业务支撑，涵盖了公告管理、材料管理、量化考核等核心功能模块。所有接口都经过精心设计，具有良好的扩展性和维护性。
