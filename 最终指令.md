# 论文写作全流程指令（Trae专用版）

本指令文件旨在指导 AI 分步骤完成系统模块划分、README 撰写、论文目录规划、字数分配及最终论文创作。请严格遵循每一步的指令文本，按顺序执行。

***

## 步骤一：生成项目的project\_rules.md 和system\_function\_module.md文件

**复制以下内容发送给 AI：**

```
你是一名全栈技术领域的专家，同时也具备学术写作顾问的能力。现在我需要你按照以下要求处理我的项目：

请充分识别当前项目代码库（包括前端和后端目录）的功能与架构。

基于对当前代码库的了解，规划项目规则，生成”project_rules.md“文件和”system_function_module.md“，”project_rules.md“文件包含项目概述、技术栈、后端配置。前端配置、项目结构、系统功能模块文档（配置"system_function_module.md"路径即可）。

其中对于“system_function_module.md”：划分标准应参考典型的软件工程论文结构，例如：用户模块、数据采集模块、预警模块、可视化模块、决策支持模块等。每个模块包括：模块概述、数据库设计、后端代码结构、前端代码结构等。系统功能模块文档还包括：系统技术架构总结、开发规范与最佳实践、测试建议等。

在生成“system_function_module.md”时，先列出你划分的模块清单，并简要说明每个模块的核心职责。列出后请暂停，等待我确认需要哪些模块。

在我确认模块划分无需修改后，请根据确认的模块清单，生成完整的”project_rules.md“文件和”system_function_module.md“文件。
```

***

## 步骤二：生成论文目录（论文目录.md）

**复制以下内容发送给 AI（需在完成步骤一并获得 project\_rules.md和system\_function\_module.md 后执行）：**

```
你是一名全栈技术领域的专家及学术写作顾问。请基于以下材料为我生成毕业论文的目录：

请仔细阅读当前系统的 #project_rules.md 和 #system_function_module.md文件内容，确保目录结构与系统实际开发模块紧密对应。

参考我提供的 #其他系统的论文目录.md 文件（如未提供，请遵循标准的计算机专业毕业论文结构：绪论、相关技术、系统分析、系统设计、系统实现、系统测试、总结）。

请生成一份 论文目录.md 文件。

撰写要求：

目录只需列出到二级标题即可。

输出内容为纯净的目录结构，无需额外的解释性文字。

标题命名需客观、严谨，避免使用口语化或文学化词汇。

符号全部使用中文格式。
```

***

## 步骤三：生成论文字数分配方案（论文字数分配方案.md）

**复制以下内容发送给 AI（基于已生成的 project\_rules.md 、system\_function\_module.md 和 论文目录.md）：**

```
你是一名学术写作规划专家。请根据以下文件为我制定详细的论文字数分配方案：

依据文件：系统的 #project_rules.md 文件、#system_function_module.md 文件和 #论文目录.md 文件。

参考文件：#其他系统的论文字数分配方案.md（如未提供，请按常规计算机毕设各部分权重分配）。

请生成 论文字数分配方案.md 文件。

方案需满足以下约束：

必须列出每个二级标题及其对应的具体字数（而非范围）。

总字数需符合【此处替换为总字数要求，如 15000 字】。

在方案末尾补充一句说明：正文段落写作时，相邻段落（非标题间隔）的字数必须相差 50 字以上，以规避 AI 规律性排布。

表达清晰，表格形式呈现更佳，符号使用中文格式。
```

***

## 步骤四：生成论文各章节创作指令（创作指令.md）

**复制以下内容发送给 AI（基于前述所有已生成文件）：**

```
你现在是一名专业的学术写作顾问，请按照以下要求生成面向其他 AI 创作助手的指令集：

请参考已准备好的全部文件：

#project_rules.md（了解系统大概情况）

#system_function_module.md （了解系统功能模块）

#参考文献.md（了解引文格式与内容）

#论文目录.md（了解论文目录）

#论文字数分配方案.md（了解论文字数分配情况）

#示例创作指令1.md（示例创作指令1，可能提供）

#示例创作指令2.md（示例创作指令2，可能提供）

请生成一个 创作指令.md 文件。该文件需要包含针对论文每一个章节（从摘要、绪论到总结）的具体创作指令。这些指令将直接交给其他写作型 AI 去执行生成论文正文。

除了摘要章节需要放在总结之后，其他章节按目录顺序排列。

每一条创作指令必须包含以下核心要求（直接写入指令中，以约束写作 AI）：

字数约束：指明该章节的建议字数。

内容锚点：指明需参考 #system_function_module.md  中的哪些功能模块（如果是技术实现章节，需明确指示“必须严格基于当前系统的实际代码逻辑写作，不得凭空虚构技术细节”）。

真实性要求：所有阐述的观点和数据必须真实有效，不得凭空虚构任何虚假情况。

风格强制要求：

确保使用清晰简洁的语言，避免冗杂的句式结构。

相邻正文段落（隔标题不算）字数相差 50 字以上（极其重要）。

段落之间不要用排比句，不要用双引号。

全部符号使用中文格式，中英文之间不要用空格隔开。

尽量减少使用“首先、其次、最后、然而、综上所述、此外、具体而言、与此同时”等过渡词。

消除任何已知的 AI 生成痕迹，避免口语化，同时避免原则性技术错误。

任何技术名称和框架名称不需要添加版本号

所有阐述的观点和数据必须真实有效，不得凭空虚构任何虚假情况。

如果需要插入系统截图，需在正文中用括号标明位置，例如【此处插入系统 XX 模块界面截图】。

示例指令可参考 #示例创作指令1.md 文件和 #示例创作指令2.md文件（如果未提供这两个文件，则参考以下示例：
1、请参考我准备好的 参考文献.md ，我需要你撰写一篇计算机专业的学术论文的摘要+四个关键词，撰写摘要时请确保使用清晰简洁的语言，避免冗杂的句式结构，以便读者能快速把握研究的核心内容和创新点，摘要应该包含研究的目的、方法、主要结果和结论，但不要过多的展开细节，并且避免使用专业术语或者缩写，确保非专业读者也能理解。）
```

***

## 步骤五：执行生成论文具体章节

**当有了** **`创作指令.md`** **后，针对每一章分别发送指令（以摘要为例）：**

```
参考前端和后端目录下的系统实际代码，依据指令文件 创作指令.md 中关于【摘要/第一章/第二章...】的要求，为我生成对应章节内容。

其他备注约束：

字数请严格参考 论文字数分配方案.md 文件。

技术描述请严格参考 #system_function_module.md  文件中划分的模块功能。

结构请符合 #论文目录.md。

写作风格强制约束（再次强调）：

相邻段落字数差 > 50 字。

无 AI 过渡词。

中文符号，无多余空格。

需结合代码逻辑，若有系统截图需说明位置。
```

***

## 步骤六：验证论文具体内容真实性

**当执行了** **`创作指令.md`** **生成论文内容后，执行指令验证论文内容的真实性：**

```
引用所有通过 #创作指令.md 生成的章节内容，通过网络搜索验证其中阐述内容的真实性

检查所有内容的真实性，所有阐述的观点和数据必须真实有效，不得凭空虚构任何虚假情况。

如果有虚假情况，需要重新生成该部分内容。

重新生成的内容需要重新验证真实性。

重点关注：
第一章、第二章、第三章

重写限制：
同一部分重写超过三次则取消重写，并暂停会话，提示用户该章节的真实性有问题。
```



## 附录：论文优化与审查指令（用于定稿阶段）

**论文写完后，可使用以下指令进行合规性检查：**

```
检查我的论文是否存在原则性错误以及严重的 AI 痕迹。AI 痕迹特指未删除的 AI 文字片段（如“当然可以”、“根据我的分析”等），而非单纯指内容的学术特征。

论文全文如下：【粘贴论文内容】

另外请帮我生成中英文摘要及关键词，字数 300 字左右，要求同样遵循前述写作风格规范（无过渡词、段落差异、中文符号）。
```
