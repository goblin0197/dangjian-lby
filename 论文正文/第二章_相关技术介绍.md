# 第二章 相关技术介绍

## 2.1 后端技术栈

### 2.1.1 Spring Boot框架

Spring Boot是Pivotal团队在Spring框架基础上推出的轻量级开发框架，其核心设计理念是通过自动配置机制和起步依赖管理大幅降低Java Web应用的开发门槛。本系统采用Spring Boot 2.7.2版本作为后端基础框架，该版本在稳定性和性能表现上经过了大量生产环境的验证。框架的自动配置功能通过`@SpringBootApplication`注解一键激活，能够根据项目classpath中引入的依赖包自动装配所需的Bean组件，开发者无需编写繁琐的XML配置文件即可快速搭建可运行的应用程序。内嵌Tomcat服务器使得项目打包为JAR文件后可直接通过`java -jar`命令启动运行，简化了部署流程。在本系统的代码实现中，`@RestController`注解标注于`com.coder.springbootinit.controller`包下的19个Controller类之上，将每个类的处理方法自动注册为RESTful风格的HTTP接口端点；`@Service`注解则应用于23个ServiceImpl实现类，标识其为业务逻辑层的核心组件供Spring IoC容器统一管理和依赖注入。Spring Boot与MyBatis-Plus、Spring Session、Spring AOP等中间件的集成均通过Maven坐标声明即可完成，起步依赖机制自动解决了版本兼容性问题。

### 2.1.2 MyBatis-Plus持久层框架

MyBatis-Plus是MyBatis的增强工具包，在保留MyBatis灵活SQL控制能力的前提下提供了大量的开箱即用功能。本系统中所有的Service接口均继承自IService泛型接口，对应的Mapper接口继承自BaseMapper泛型基类，由此获得了单表CRUD操作的默认实现而无须手写SQL语句。以用户查询场景为例，当需要按角色和所属组织进行分页筛选时，通过LambdaQueryWrapper条件构造器链式调用`.eq(User::getUserRole, role)`和`.eq(User::getOrgId, orgId)`即可构建动态查询条件，底层由MyBatis-Plus自动拼接为标准的WHERE子句并执行分页查询。全局配置层面，系统在application.yml中设置了主键ID的自增策略（IdType.AUTO）和逻辑删除字段isDelete（删除标记值设为1），确保数据软删除的一致性。23个Mapper接口覆盖了user、organization、activity、notice、material_submission、user_quantify等全部业务实体，配合Mapper XML映射文件中定义的多表关联查询语句，构成了完整的数据访问层。

### 2.1.3 MySQL数据库

MySQL是目前应用最为广泛的开源关系型数据库管理系统，以其高可靠性、事务支持能力和成熟的索引优化机制成为Web应用后端存储的首选方案之一。本系统使用MySQL 8.x版本创建名为party_building的业务数据库，采用utf8mb4字符集以完整支持中文及特殊Unicode字符的存储需求。数据库中包含user用户表存储账号密码和角色信息、organization组织表维护树形层级结构、activity活动主表及其关联的activity_enroll报名记录表、notice公告表及其扩展的附件表和阅读记录表、material_submission材料提交表记录审核流转状态、user_quantify和org_quantify两张量化统计表存储每日计算的指标数据、development_stage发展阶段表跟踪入党全流程、trainer_relation培养人关系表以及org_relation_transfer组织关系转移表等核心业务表。InnoDB存储引擎的事务特性保证了多表操作的数据一致性要求，例如活动报名时同时写入报名记录和更新活动当前人数的操作被包裹在同一事务中执行。

### 2.1.4 Redis缓存技术

Redis是一个基于内存的高性能键值对数据存储系统，在本系统中承担分布式会话管理的职责。前后端分离架构下，浏览器每次请求API接口时需要携带身份凭证以证明登录状态，传统基于服务器内存的Session机制在多实例部署场景下存在数据不同步的问题。本系统引入Spring Session框架将会话数据持久化至Redis服务中，配置连接地址为localhost:6379并指定database=2作为专用数据库编号，会话过期时间设定为30天（2592000秒）。用户调用`UserController.userLogin()`完成账号密码验证后，`UserServiceImpl.userLogin()`方法内部通过`request.getSession().setAttribute(USER_LOGIN_STATE, user)`将登录用户对象写入HttpSession，Spring Session框架自动将该Session数据序列化持久化至Redis存储并以Set-Cookie响应头的形式将JSESSIONID返回给浏览器保存；后续请求携带该Cookie时，框架从Redis中还原Session数据，各Controller方法可通过`userService.getLoginUser(request)`从Session中取出当前登录用户的身份信息。这种方案使得即使后端服务重启也不会丢失已登录用户的会话状态。

## 2.2 前端技术栈

### 2.1.1 Vue.js框架

Vue.js是一套用于构建用户界面的渐进式JavaScript框架，其核心设计思想是将视图层的状态变化抽象为响应式的数据驱动模型。本系统前端采用Vue 3.x版本开发，该版本将底层数据响应机制从Vue 2的Object.defineProperty替换为ES6 Proxy代理对象，实现了对数组下标赋值和对象属性动态添加等边缘场景的精确拦截检测，减少了响应式失效的潜在风险。组合式API（Composition API）允许开发者将同一功能的逻辑代码集中组织在setup函数内部，相较于选项式API按生命周期钩子分散编排的方式具有更好的代码复用性和类型推导能力。前端工程目录下的src/views文件夹按照功能模块划分存放了ActivityView.vue活动管理页面、OrganizationView.vue组织管理页面、QuantifyView.vue个人统计页面等多个单文件组件，每个组件通过<script setup>语法糖声明响应式变量和业务方法。Vuex 4.0状态管理库维护了全局的用户登录态信息（store/user.ts），路由守卫机制则位于access/目录下根据用户角色判断目标页面的访问权限并在无权限时重定向至登录页。

### 2.1.2 Arco Design Vue组件库

Arco Design Vue是字节跳动开源的基于Vue 3的企业级UI组件库，为本系统前端界面提供了完整的交互控件支撑。在各管理页面的数据列表展示区域广泛使用了a-table表格组件，其内置的分页器、排序触发器和列筛选功能满足了大部分CRUD场景的展示需求。表单提交操作统一封装在a-form表单容器中，配合a-input输入框、a-select下拉选择器、a-date-picker日期选择器和a-upload文件上传组件构建出完整的录入界面。弹窗类交互通过a-modal模态框组件实现，用于新增编辑操作的表单弹出和信息确认提示。左侧导航菜单采用a-menu组件渲染，结合Vue Router的路由配置项实现菜单项与页面路径的自动高亮联动。各页面通过`import ... from "@arco-design/web-vue/es/..."`方式引入组件和工具方法（如message消息提示、Icon系列图标组件等），确保了UI交互风格的一致性。除Arco Design Vue外，系统package.json中还引入了Ant Design Vue 4.2.6作为备选组件库依赖。

### 2.1.3 ECharts可视化库

ECharts是由百度开源的数据可视化图表库，具备跨平台渲染能力和丰富的图表类型支持。本系统在量化统计模块中深度集成了ECharts 6.0版本，用于将聚合计算后的党建指标数据转化为直观的图形化呈现形式。个人统计页面QuantifyView.vue中配置了折线图组件展示用户近期的活动参与度趋势变化，横轴为统计日期、纵轴为参与率百分比数值，多条折线可叠加对比签到率和材料数的变化轨迹。柱状图组件用于呈现各类活动类型的参与分布情况，便于用户了解自身在不同类别党建活动中的投入程度。管理员统计面板QuantifyManagerView.vue中则使用了饼图组件展示各党组织活动数量占比，以及堆叠柱状图对比不同组织的平均签到率和平均参与人数指标。图表的数据来源分别为/api/quantify/user/{userId}和/api/quantify/org/{orgId}两个后端接口返回的JSON格式统计数据，前端在onMounted生命周期钩子中发起请求获取数据后传入ECharts实例的setOption方法完成图表渲染。

## 2.3 开发工具与环境

后端代码的开发环境选用IntelliJ IDEA集成开发环境，该工具对Java项目的Maven依赖管理、代码智能补全、断点调试和热重载部署提供了完善的支持。前端代码采用Visual Studio Code编辑器进行开发，借助Vetur和TypeScript插件获得Vue单文件组件的语法高亮和类型检查能力。版本控制工具使用Git对全部源代码进行变更追踪和分支管理，代码仓库托管于远程Git服务平台以便于多人协作开发和版本回溯。后端工程的构建和依赖解析由Apache Maven负责，pom.xml文件中声明了Spring Boot 2.7.2父POM以及MyBatis-Plus 3.5.2、Knife4j 4.4.0、EasyExcel 3.1.1等核心依赖坐标。前端工程基于Vue CLI 5.x脚手架创建，package.json中配置了Vue Router 4.0.3、Vuex 4.0.0、Ant Design Vue 4.2.6等生产依赖以及TypeScript编译器和ESLint代码检查工具等开发依赖。Java运行时环境采用JDK 11版本，Node.js运行时环境用于执行前端构建脚本和开发服务器。

## 2.4 本章小结

本章介绍了智慧党建管理系统所采用的全部核心技术栈和开发工具。后端以Spring Boot 2.7.2为基础框架整合MyBatis-Plus 3.5.2持久层、MySQL 8.x关系型数据库和Redis缓存中间件构成稳定可靠的服务端架构；前端基于Vue 3.x响应式框架搭配Arco Design Vue企业级组件库和ECharts可视化引擎构建交互友好的用户界面。上述技术均为业界成熟的开源方案，文档资料丰富、社区生态活跃，能够有效支撑本系统的开发和长期运维需求。
