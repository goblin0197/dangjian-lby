# 量化数据自动生成实现计划

## 1. 现状分析

### 1.1 当前实现
- **量化数据（QuantifyData）**：通过接口手动添加（QuantifyDataController中有addData和batchAddData接口）
- **量化指标（QuantifyIndicator）**：用于定义指标规则，但当前没有根据指标自动生成数据的逻辑
- **定时任务（QuantifyScheduledTask）**：生成user_quantify和org_quantify表的数据，不是quantify_data表的数据

### 1.2 存在问题
- 量化数据需要手动添加，不符合业务逻辑
- 量化指标与量化数据之间的关联未充分利用
- 缺少根据量化指标自动生成数据的机制

## 2. 实现目标

将量化数据的生成方式从手动添加改为根据量化指标自动生成，具体目标：
1. 修改定时任务，使其根据量化指标生成量化数据
2. 为QuantifyDataService添加根据指标生成数据的方法
3. 移除或禁用手动添加量化数据的接口
4. 确保生成的数据与量化指标正确关联

## 3. 实现步骤

### 3.1 扩展QuantifyDataService接口
- 添加`generateDataByIndicator`方法：根据单个指标生成数据
- 添加`generateAllDataByIndicators`方法：根据所有启用的指标生成数据

### 3.2 实现QuantifyDataServiceImpl
- 实现上述新增方法
- 根据量化指标的规则计算对应的数据值
- 支持不同维度（组织/个人）的数据生成
- 支持不同统计周期的数据生成

### 3.3 修改QuantifyScheduledTask
- 在`generateQuantifyDataDaily`方法中添加调用`QuantifyDataService.generateAllDataByIndicators`的逻辑
- 确保定时任务能够根据量化指标自动生成数据

### 3.4 更新QuantifyDataController
- 移除或禁用`addData`和`batchAddData`接口
- 添加触发数据生成的接口（可选）

### 3.5 数据模型优化
- 确保QuantifyData实体与QuantifyIndicator实体的关联正确
- 可能需要调整字段以更好地支持自动生成逻辑

## 4. 技术实现细节

### 4.1 量化数据生成逻辑
1. 获取所有启用的量化指标
2. 对每个指标，根据其维度（组织/个人）和规则计算数据
3. 为每个目标（组织/用户）生成对应的数据记录
4. 按统计周期存储数据

### 4.2 指标规则解析
- 解析量化指标的rule字段，提取计算规则
- 根据规则从相关表（活动、材料等）获取原始数据
- 计算并生成量化数据

### 4.3 性能考虑
- 批量处理数据，减少数据库操作次数
- 考虑使用多线程提高生成效率
- 实现增量更新，只处理变化的数据

## 5. 风险评估

### 5.1 潜在风险
- 量化指标规则解析复杂度高
- 数据生成过程可能影响系统性能
- 生成的数据量较大时可能导致数据库压力

### 5.2 缓解措施
- 实现规则解析的错误处理机制
- 控制数据生成的频率和范围
- 优化数据库查询和写入操作

### 6.2 性能测试
- 测试大量数据生成的性能
- 测试定时任务的执行时间

## 7. 预期效果

- 量化数据自动根据量化指标生成，无需手动添加
- 数据生成过程自动化，减少人工操作
- 数据与指标的关联更加清晰
- 系统维护成本降低
