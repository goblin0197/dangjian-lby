# 材料管理系统接口设计

## 1. 材料模板管理接口

| 接口名称                              | URL                                      | 方法     | 作用               |
|-----------------------------------|------------------------------------------|--------|------------------|
| getMaterialTemplateList           | /api/material/template/list              | GET    | 获取材料模板列表，支持分页和筛选 |
| addMaterialTemplate               | /api/material/template/add               | POST   | 新增材料模板，包括上传文件    |
| updateMaterialTemplate            | /api/material/template/update            | PUT    | 更新材料模板信息         |
| deleteMaterialTemplate            | /api/material/template/delete            | DELETE | 删除单个材料模板         |
| batchDeleteMaterialTemplates      | /api/material/template/batchDelete       | DELETE | 批量删除材料模板         |
| toggleMaterialTemplateStatus      | /api/material/template/toggleStatus      | PUT    | 切换模板启用/停用状态      |
| batchToggleMaterialTemplateStatus | /api/material/template/batchToggleStatus | PUT    | 批量切换模板状态         |
| previewMaterialTemplate           | /api/material/template/preview           | GET    | 预览模板文件内容         |
| downloadMaterialTemplate          | /api/material/template/download          | GET    | 下载模板文件           |
| uploadMaterialTemplateFile        | /api/upload/template                     | POST   | 上传模板文件           |

## 5. 接口参数说明

### 5.1 模板管理接口参数

**获取模板列表**

- 参数：stage（发展阶段）、type（材料类型）、status（模板状态）、page（页码）、pageSize（每页数量）

**新增/编辑模板**

- 参数：name（模板名称）、stage（发展阶段）、type（材料类型）、status（模板状态）、file（文件）、remark（说明）

**删除/批量删除模板**

- 参数：id（模板ID）或 ids（模板ID列表）

### 5.2 材料提交接口参数

**获取材料列表**

- 参数：timeRange（时间范围）、orgLevel（组织层级）、stage（发展阶段）、submitStatus（提交状态）、page（页码）、pageSize（每页数量）

**提交材料**

- 参数：userId（用户ID）、stage（发展阶段）、materialName（材料名称）、file（文件）、remark（备注）

**审核/退回材料**

- 参数：id（提交ID）、auditResult（审核结果）、auditOpinion（审核意见）

### 5.3 归档查询接口参数

**获取归档列表**

- 参数：userName（姓名）、materialName（材料名称）、stage（发展阶段）、archiveTime（归档时间范围）、page（页码）、pageSize（每页数量）

**归档材料**

- 参数：submissionId（提交ID）、archiveRemark（归档备注）

**批量导出归档**

- 参数：ids（归档ID列表）

## 6. 响应格式

所有接口响应格式统一为：

```json
{
  "code": 0,
  "data": {},
  "message": "success"
}
```

## 7. 权限控制

- **管理员**：所有接口可访问
- **培养联系人**：可访问材料提交、审核相关接口，查看教师支部归档材料
- **普通党员**：可访问材料提交相关接口，查看本人归档材料
