# 公告管理系统接口设计

## 1. 公告管理接口

| 接口名称 | URL | 方法 | 作用 |
|---------|-----|------|------|
| getAnnouncementList | /api/announcement/list | GET | 获取公告列表，支持分页和筛选 |
| getAnnouncementDetail | /api/announcement/detail | GET | 获取公告详细信息 |
| addAnnouncement | /api/announcement/add | POST | 新增公告，包括富文本内容和附件 |
| updateAnnouncement | /api/announcement/update | PUT | 更新公告信息 |
| deleteAnnouncement | /api/announcement/delete | DELETE | 删除公告 |
| batchDeleteAnnouncements | /api/announcement/batchDelete | DELETE | 批量删除公告 |
| publishAnnouncement | /api/announcement/publish | PUT | 发布公告 |
| offlineAnnouncement | /api/announcement/offline | PUT | 下架公告 |
| republishAnnouncement | /api/announcement/republish | PUT | 重新发布公告 |
| topAnnouncement | /api/announcement/top | PUT | 设置公告置顶状态 |

## 2. 公告阅读接口

| 接口名称 | URL | 方法 | 作用 |
|---------|-----|------|------|
| markAnnouncementAsRead | /api/announcement/markRead | POST | 标记公告为已读 |
| getUnreadCount | /api/announcement/unreadCount | GET | 获取未读公告数量 |
| getReadList | /api/announcement/readList | GET | 获取用户已读公告列表 |
| incrementReadCount | /api/announcement/incrementReadCount | PUT | 增加公告阅读量 |

## 3. 公告附件接口

| uploadAnnouncementAttachment | /api/upload/notice/attachment | POST | 上传公告附件 |
| uploadAnnouncementImage | /api/upload/notice/image | POST | 上传富文本编辑器图片 |
| downloadAnnouncementAttachment | /api/announcement/attachment/download | GET | 下载公告附件 |
| previewAnnouncementAttachment | /api/announcement/attachment/preview | GET | 预览公告附件 |
| deleteAnnouncementAttachment | /api/announcement/attachment/delete | DELETE | 删除公告附件 |

## 4. 公告分享接口

| 接口名称 | URL | 方法 | 作用 |

## 6. 接口参数说明

### 6.1 公告管理接口参数

**获取公告列表**
- 参数：type（公告类型）、status（公告状态）、timeRange（时间范围）、keyword（关键词）、page（页码）、pageSize（每页数量）

**新增/编辑公告**
- 参数：title（标题）、type（类型）、content（内容）、isTop（是否置顶）、validStart（生效开始时间）、validEnd（生效结束时间）、attachments（附件列表）

**发布/下架公告**
- 参数：id（公告ID）

### 6.2 公告阅读接口参数

**标记已读**
- 参数：announcementId（公告ID）、userId（用户ID）

**获取未读数量**
- 参数：userId（用户ID）

### 6.3 公告附件接口参数

**上传附件**
- 参数：file（文件）、announcementId（公告ID）

**下载/预览附件**
- 参数：attachmentId（附件ID）

### 6.4 公告分享接口参数

**生成分享链接**
- 参数：announcementId（公告ID）、expireTime（过期时间）

**访问分享链接**
- 参数：shareCode（分享码）

## 7. 响应格式

所有接口响应格式统一为：

```json
{
  "code": 0,
  "data": {},
  "message": "success"
}
```

## 8. 权限控制

- **管理员**：所有接口可访问
- **培养联系人**：可访问公告列表、详情、标记已读接口
- **普通党员**：可访问公告列表、详情、标记已读接口
