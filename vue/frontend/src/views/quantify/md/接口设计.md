# 量化管理功能接口设计

## 1. 量化指标管理接口

### 1.1 获取量化指标列表

- **接口名称**：getQuantifyIndicators
- **URL**：/api/quantify/indicator/list
- **方法**：GET
- **参数**：
    - indicator：统计指标类型（可选）
    - orgLevel：组织层级（可选）
    - status：状态（可选）
    - current：页码（可选，默认1）
    - pageSize：每页大小（可选，默认10）
- **返回值**：
    - code：状态码
    - data：指标列表
    - message：消息
- **作用**：获取量化指标列表，支持筛选和分页

### 1.2 新增量化指标

- **接口名称**：addQuantifyIndicator
- **URL**：/api/quantify/indicator/add
- **方法**：POST
- **参数**：
    - name：指标名称
    - rule：统计规则
    - dimension：统计维度
    - orgLevel：适用组织层级
    - status：状态
- **返回值**：
    - code：状态码
    - data：新增的指标
    - message：消息
- **作用**：新增量化指标

### 1.3 编辑量化指标

- **接口名称**：updateQuantifyIndicator
- **URL**：/api/quantify/indicator/update
- **方法**：POST
- **参数**：
    - id：指标ID
    - name：指标名称
    - rule：统计规则
    - dimension：统计维度
    - orgLevel：适用组织层级
    - status：状态
- **返回值**：
    - code：状态码
    - data：更新后的指标
    - message：消息
- **作用**：编辑量化指标

### 1.4 删除量化指标

- **接口名称**：deleteQuantifyIndicator
- **URL**：/api/quantify/indicator/delete
- **方法**：POST
- **参数**：
    - id：指标ID
- **返回值**：
    - code：状态码
    - data：删除结果
    - message：消息
- **作用**：删除量化指标

### 1.5 批量删除量化指标

- **接口名称**：batchDeleteQuantifyIndicators
- **URL**：/api/quantify/indicator/batchDelete
- **方法**：POST
- **参数**：
    - ids：指标ID列表
- **返回值**：
    - code：状态码
    - data：删除结果
    - message：消息
- **作用**：批量删除量化指标

### 1.6 切换指标状态

- **接口名称**：toggleQuantifyIndicatorStatus
- **URL**：/api/quantify/indicator/toggleStatus
- **方法**：POST
- **参数**：
    - id：指标ID
    - status：新状态
- **返回值**：
    - code：状态码
    - data：切换结果
    - message：消息
- **作用**：切换量化指标的启用/停用状态

### 1.7 导入量化指标模板

- **接口名称**：importQuantifyIndicatorTemplate
- **URL**：/api/quantify/indicator/import
- **方法**：POST
- **参数**：
    - file：Excel文件
- **返回值**：
    - code：状态码
    - data：导入结果
    - message：消息
- **作用**：通过Excel模板导入量化指标

## 2. 量化数据统计接口

### 2.1 获取核心指标数据

- **接口名称**：getCoreIndicators
- **URL**：/api/quantify/data/core
- **方法**：GET
- **参数**：
    - timeRange：时间范围
    - orgLevel：组织层级
    - dimension：统计维度
- **返回值**：
    - code：状态码
    - data：核心指标数据
    - message：消息
- **作用**：获取核心指标数据（活动参与率、签到率、材料完成率）

### 2.2 获取组织维度统计数据

- **接口名称**：getOrganizationStatistics
- **URL**：/api/quantify/data/organization
- **方法**：GET
- **参数**：
    - timeRange：时间范围
    - orgLevel：组织层级
    - indicator：统计指标
- **返回值**：
    - code：状态码
    - data：组织统计数据
    - message：消息
- **作用**：获取组织维度的统计数据

### 2.3 获取个人维度统计数据

- **接口名称**：getPersonalStatistics
- **URL**：/api/quantify/data/personal
- **方法**：GET
- **参数**：
    - timeRange：时间范围
    - orgLevel：组织层级
    - indicator：统计指标
- **返回值**：
    - code：状态码
    - data：个人统计数据
    - message：消息
- **作用**：获取个人维度的统计数据

### 2.4 获取数据详情

- **接口名称**：getQuantifyDataDetail
- **URL**：/api/quantify/data/detail
- **方法**：GET
- **参数**：
    - id：数据ID
    - dimension：统计维度
- **返回值**：
    - code：状态码
    - data：数据详情
    - message：消息
- **作用**：获取量化数据的详细信息

### 2.5 导出统计报表

- **接口名称**：exportQuantifyReport
- **URL**：/api/quantify/data/export
- **方法**：GET
- **参数**：
    - timeRange：时间范围
    - orgLevel：组织层级
    - dimension：统计维度
    - indicator：统计指标
- **返回值**：
    - Excel文件
- **作用**：导出统计报表为Excel格式

## 3. 权限控制接口

### 3.1 获取用户权限

- **接口名称**：getUserPermission
- **URL**：/api/quantify/permission
- **方法**：GET
- **参数**：无
- **返回值**：
    - code：状态码
    - data：用户权限信息
    - message：消息
- **作用**：获取当前用户的权限信息，用于控制页面功能的显示和操作权限

## 4. 接口调用说明

- 所有接口均需要在请求头中携带认证信息（如token）
- 接口返回格式统一为：{"code": 0, "data": {}, "message": "success"}
- 当code为0时表示请求成功，非0时表示请求失败，message字段会包含失败原因
- 接口参数中的可选字段如未提供，将使用默认值
- 导出接口返回的是文件流，需要特殊处理
