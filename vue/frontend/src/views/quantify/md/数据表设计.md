# 量化管理功能数据表设计

## 1. 量化指标表（quantifyIndicator）

```sql
create table if not exists quantifyIndicator
(
    id           bigint auto_increment comment 'id' primary key,
    name         varchar(256)                           not null comment '指标名称',
    rule         varchar(512)                           not null comment '统计规则',
    dimension    varchar(64)                            not null comment '统计维度：organization-组织/personal-个人/both-两者',
    orgLevel     varchar(512)                           not null comment '适用组织层级，JSON格式存储',
    status       varchar(32)  default 'enable'           not null comment '状态：enable-启用/disable-停用',
    createTime   datetime     default CURRENT_TIMESTAMP not null comment '创建时间',
    updateTime   datetime     default CURRENT_TIMESTAMP not null on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete     tinyint      default 0                 not null comment '是否删除',
    index idx_dimension (dimension),
    index idx_status (status)
) comment '量化指标表' collate = utf8mb4_unicode_ci;
```

## 2. 量化数据记录表（quantifyData）

```sql
create table if not exists quantifyData
(
    id               bigint auto_increment comment 'id' primary key,
    indicatorId      bigint                               not null comment '指标ID',
    targetId         bigint                               not null comment '统计对象ID（组织ID或用户ID）',
    targetType       varchar(32)                          not null comment '统计对象类型：organization-组织/user-用户',
    period           varchar(32)                          not null comment '统计周期（如：2025.01-03）',
    value            decimal(10,2)                        not null comment '统计值',
    activityRate     decimal(10,2)    default 0           not null comment '活动参与率',
    signRate         decimal(10,2)    default 0           not null comment '签到率',
    materialRate     decimal(10,2)    default 0           not null comment '材料完成率',
    activityCount    int              default 0           not null comment '参与活动数',
    unfinishedMaterial int         default 0           not null comment '未完成材料数',
    personCount      int              default 0           not null comment '支部人数（仅组织维度）',
    averageRate      decimal(10,2)    default 0           not null comment '平均参与率（仅组织维度）',
    createTime       datetime         default CURRENT_TIMESTAMP not null comment '创建时间',
    updateTime       datetime         default CURRENT_TIMESTAMP not null on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete         tinyint          default 0           not null comment '是否删除',
    index idx_indicatorId (indicatorId),
    index idx_targetId (targetId),
    index idx_targetType (targetType),
    index idx_period (period)
) comment '量化数据记录表' collate = utf8mb4_unicode_ci;
```

## 3. 数据关联关系

### 3.1 量化指标表与量化数据记录表
- 量化指标表（quantifyIndicator）的id字段与量化数据记录表（quantifyData）的indicatorId字段关联
- 一个量化指标可以对应多条量化数据记录

### 3.2 量化数据记录表与其他表
- 当targetType为'organization'时，targetId关联组织表（organization）的id字段
- 当targetType为'user'时，targetId关联用户表（user）的id字段

## 4. 数据存储说明

- **量化指标表**：存储系统中所有的量化指标配置，包括指标名称、统计规则、适用维度等
- **量化数据记录表**：存储具体的量化数据，按统计周期、统计对象和指标类型进行存储
- **数据更新**：量化数据应定期（如每天或每周）更新，确保数据的及时性和准确性
- **数据归档**：对于历史数据，可考虑进行归档处理，以提高查询性能
